Kayako Messenger Identification

A simple HTML page which implements Kayako Messenger Identification and allows users to start chat without providing an email address. It's very useful for projects where clients want to integrate Kayako Chat with their websites and authenticate user from their system. It also implements an additional feature of passing custom information in Kayako Chat which is not possible with the default product feature set.

Kayako SSO JWT

This project implements brand specific SSO so that customers are redirected to the assigned brand instead of the primary/default brand in Kayako. The SSO app is hosted in Heroku and uses a PHP stack to complete the SSO JWT authentication. It works by identifying the domain in the return URL when an end user invokes the SSO request.
